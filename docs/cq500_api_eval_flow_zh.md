# CQ500 API 评估流程图

## 概述

本文档描述 `scripts/run_cq500_api_eval.py` 的完整执行流程。
该脚本使用视觉语言模型（VLM）对 CQ500 头部 CT 数据集进行诊断评估，
对比模型在**原始图像**（clean）和**金属伪影图像**（degraded）上的诊断性能差异。

## 关键设计决策

1. **多数投票**: 3 位放射科医生标注每个病例；≥2 人同意则标签为 1
2. **切片采样**: 从序列中间 50% 区域选取最多 5 张切片（诊断价值最高的区域）
3. **双路评估**: 同时测试 `clean`（原图）和 `degraded`（金属伪影图），衡量伪影鲁棒性
4. **CT 窗位窗宽**: 使用脑窗 (center=40 HU, width=80 HU) 以获得最佳软组织对比度
5. **速率限制**: API 调用间可配置延迟，遵守服务商限额

## 输出指标含义速查

| 指标 | 含义 | 说明 |
|------|------|------|
| accuracy | 平均准确率 | 每个标签 (TP+TN)/总数 的均值 |
| macro_F1 | 宏平均 F1 | 各标签 F1 的算术平均，对稀有类敏感 |
| micro_F1 | 微平均 F1 | 全局 TP/FP/FN 计算，对高频类权重大 |
| auroc | ROC 曲线下面积 | 基于置信度的排序能力指标（需 sklearn） |
| latency | 平均延迟 | 单次 API 调用的平均响应时间 |
| valid / failed | 有效/失败数 | API 调用成功/解析成功的计数 |
| Degraded drop | 退化降幅 | (clean_acc - degraded_acc) / clean_acc × 100% |
